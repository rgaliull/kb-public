# Сравнение self-hosted password managers для команд

> Выбор менеджера паролей для команды — компромисс между безопасностью, удобством шаринга и администрирования. Ниже сравнение open-source решений по ключевым критериям.

---

## Критерии оценки

- **Web-интерфейс** — наличие и качество
- **RBAC** — ролевое управление доступом (роли, группы, права)
- **Шаринг одного пароля** — можно ли поделиться конкретной записью без создания коллекций/папок
- **E2E-шифрование** — знает ли сервер содержимое паролей
- **Сложность установки** — Docker, LAMP, зависимости
- **Лицензия и стоимость**

---

## Passbolt

Самый зрелый командный менеджер паролей с E2E-шифрованием на OpenPGP и прямым шарингом.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐ Отличный — современный React |
| **RBAC** | ✅ Роли, группы, детальные права доступа |
| **Шаринг 1 пароля** | ✅ Прямой — пользователю или группе |
| **E2E-шифрование** | ✅ OpenPGP (приватный ключ у пользователя) |
| **Установка** | Docker, deb-пакет, Kubernetes |
| **Стек** | PHP/CakePHP, MySQL/PostgreSQL |
| **MFA** | TOTP, YubiKey, Duo |
| **LDAP/AD** | Pro/Enterprise |
| **Лицензия** | AGPL (Community), Pro/Enterprise — платные |
| **API** | ✅ REST API |

**Итог:** лучший выбор для командной работы. Единственный минус — часть функций (LDAP, группы) платная.

---

## Psono

Ближайший конкурент Passbolt — клиентское E2E-шифрование, современный интерфейс, прямой шаринг.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐ Отличный — современный |
| **RBAC** | ✅ Роли, группы, детальные права доступа |
| **Шаринг 1 пароля** | ✅ Прямой |
| **E2E-шифрование** | ✅ Клиентское шифрование |
| **Установка** | Docker |
| **Стек** | Python/Django, PostgreSQL |
| **MFA** | TOTP, YubiKey, Duo |
| **LDAP/SAML/OIDC** | Enterprise Edition |
| **Лицензия** | Apache 2.0 (Community), Enterprise — платная |
| **API** | ✅ |

**Итог:** достойная альтернатива Passbolt с сопоставимым уровнем безопасности. Минус — Enterprise-фичи платные, сообщество меньше.

---

## Padloc

Современный менеджер паролей для команд и индивидуального использования. PWA-клиент, E2E-шифрование, Docker.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐ Отличный — современный PWA |
| **RBAC** | ✅ Роли, группы, организации |
| **Шаринг 1 пароля** | ✅ Прямой — через vaults и шаринг |
| **E2E-шифрование** | ✅ Клиентское (SRP + AES) |
| **Установка** | Docker (монорепо, собирается из исходников) |
| **Стек** | Node.js/TypeScript |
| **MFA** | ✅ |
| **LDAP/SAML** | Через внешние интеграции |
| **Лицензия** | AGPLv3 |
| **GitHub** | ⭐ ~2,900 |

**Итог:** перспективный, но установка сложнее — нет готового Docker-образа, нужно собирать из исходников. Сообщество меньше, чем у Passbolt/Psono.

## Vaultwarden / Bitwarden

Vaultwarden — неофициальный легковесный сервер (Rust), совместимый с клиентами Bitwarden. Bitwarden — официальный сервер (.NET + SQL Server).

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐⭐ Нормальный — функциональный, но UX спорный |
| **RBAC** | ✅ Организации, коллекции, роли (Owner/Admin/Manager/User) |
| **Шаринг 1 пароля** | ❌ **Только через коллекцию** — создать коллекцию → положить пароль → поделиться коллекцией |
| **E2E-шифрование** | ✅ |
| **Установка** | Vaultwarden: Docker (~20MB RAM) / Bitwarden: Docker (тяжёлый, SQL Server) |
| **Стек** | Vaultwarden: Rust/SQLite / Bitwarden: .NET/SQL Server |
| **MFA** | ✅ (через клиенты Bitwarden) |
| **LDAP** | Через отдельный LDAP Connector |
| **Лицензия** | Vaultwarden: GPLv3 / Bitwarden сервер: AGPL |

**Итог:** отличный *личный* менеджер, но для команд — неудобен. Механика коллекций делает простой шаринг одного пароля громоздким. Если нужно поделиться одним доступом — это лишние 3-4 клика и создание сущности-коллекции.

---

## Teampass

PHP-менеджер с фокусом на командную работу. Минус — серверное шифрование.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐⭐ Устаревший — PHP/jQuery |
| **RBAC** | ✅ Роли (Admin/Manager/User/Read-only), группы, деревья папок |
| **Шаринг 1 пароля** | ✅ Через папки и права доступа |
| **E2E-шифрование** | ❌ **Нет** — один симметричный ключ на сервере. Сервер может читать пароли |
| **Установка** | LAMP-стек или Docker |
| **Стек** | PHP, MySQL |
| **MFA** | Google 2FA, Duo |
| **LDAP** | ✅ |
| **Лицензия** | GPLv3 |

**Итог:** функциональный, но компромиссный вариант. Главный минус — серверное шифрование: администратор сервера технически может прочитать все пароли. Интерфейс морально устарел.

---

## SysPass

PHP-менеджер, похож на Teampass, но с чуть более современным интерфейсом.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐⭐ Нормальный — современнее Teampass |
| **RBAC** | ✅ Роли, группы, профили доступа |
| **Шаринг 1 пароля** | ✅ Через группы и права |
| **E2E-шифрование** | ❌ **Нет** — мастер-ключ на сервере |
| **Установка** | LAMP-стек или Docker |
| **Стек** | PHP, MySQL |
| **MFA** | ✅ |
| **LDAP** | ✅ |
| **Лицензия** | GPLv3 |

**Итог:** чуть приятнее Teampass визуально, но та же проблема — серверное шифрование. Для команд, где безопасность не критична — рабочий вариант.

## Passwordcockpit

Лёгкий менеджер для команд — PHP, Docker, бесплатный, BSD-3.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐⭐ Нормальный — чистый, но без изысков |
| **RBAC** | ✅ Роли, группы |
| **Шаринг 1 пароля** | ✅ Через группы и права |
| **E2E-шифрование** | ⚠️ Частичное (AES на сервере + HTTPS) |
| **Установка** | Docker |
| **Стек** | PHP/JavaScript, MySQL |
| **MFA** | ⚠️ Базовая |
| **LDAP** | ✅ |
| **Лицензия** | BSD-3-Clause |

**Итог:** простой и бесплатный вариант для небольших команд. Минус — маленькое сообщество (⭐ ~140), ограниченная документация.

## AliasVault

Password manager со встроенным генератором email-алиасов (защита от утечек через уникальные email на каждый сервис).

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐⭐ Нормальный |
| **RBAC** | ⚠️ Ограниченный — проект молодой, фокус на индивидуальное использование |
| **Шаринг 1 пароля** | ⚠️ Базовый шаринг |
| **E2E-шифрование** | ✅ |
| **Установка** | Docker |
| **Стек** | .NET, PostgreSQL |
| **Лицензия** | MIT |

**Итог:** интересная идея со встроенными email-алиасами, но для командной работы пока сыроват.

## HashiCorp Vault

Secrets management для инфраструктуры. Не классический password manager, но мощный инструмент для централизованного хранения секретов, токенов, сертификатов и паролей.

| Критерий | Оценка |
|---|---|
| **Web UI** | ⭐⭐ Нормальный — есть, но функционально ограничен, основная работа через CLI/API |
| **RBAC** | ✅ Политики, пути, токены — очень гибкая система прав |
| **Шаринг 1 пароля** | ✅ Через KV-хранилище + политики доступа |
| **E2E-шифрование** | ❌ Серверное — Vault шифрует данные в storage, но имеет к ним доступ |
| **Установка** | Бинарник, Docker, Helm (K8S). Production-развёртывание сложное (HA, unseal, бэкапы) |
| **Стек** | Go, встроенное хранилище (Raft) или Consul/etcd |
| **MFA** | ✅ (TOTP, Okta, Duo, LDAP, и др.) |
| **LDAP/SAML/OIDC** | ✅ Встроенная поддержка |
| **Лицензия** | BSL (Community — бесплатно, Enterprise — платно) |
| **GitHub** | ⭐ ~35,700 |
| **API** | ✅ REST API — основной способ взаимодействия |

**Итог:** мощнейший инструмент, но **для другого сценария**. Vault создан для инфраструктурных секретов (API-ключи, токены БД, TLS-сертификаты), а не для пользовательских паролей. Нет браузерного расширения для автозаполнения логинов на сайтах. Администрирование требует серьёзной экспертизы. **Стоит выбрать, если команда уже использует Vault для инфраструктуры** — тогда логично хранить и пароли там же. Для чисто пользовательских паролей — Passbolt/Psono удобнее.

## KeePass + Web-интерфейсы

KeePass — формат .kdbx-файлов, де-факто стандарт для локального хранения. Есть несколько web-обёрток:

| Решение | Описание | Подходит для команд? |
|---|---|---|
| **KeeWeb** | Красивый web-интерфейс к KeePass-файлам | ❌ Нет RBAC — все знают мастер-пароль |
| **Pleasant Password Server** | Коммерческий сервер KeePass с web-интерфейсом | ✅ RBAC, AD/LDAP, но платный |
| **KeePass + WebDAV** | Синхронизация .kdbx через WebDAV-сервер | ❌ Конфликты при одновременном редактировании |

---

## Что не вошло в сравнение

| Решение | Причина исключения |
|---|---|
| **Buttercup** | Проект закрыт (2024). Репозитории переходят в архив |
| **LessPass** | Stateless — генерирует пароли детерминированно, не хранит их. Не подходит для командного шаринга |
| **PassIt** | Нет админ-интерфейса, только базовый шаринг по группам |
| **gopass** | CLI-first, ⭐~6,900. Git + GPG, отличная поддержка команд. **Нет полноценного web-интерфейса** — только browser-расширения. Если CLI допустим — лучший выбор для git-based команд |

## Итоговый вердикт

- **Passbolt** — лучший выбор для командной работы. Зрелый, безопасный, удобный шаринг. Если уже используете — оставайтесь на нём.
- **Psono** — полноценная альтернатива Passbolt с сопоставимым уровнем безопасности и удобства.
- **Padloc** — перспективный, но требует сборки из исходников, сообщество меньше.
- **Vaultwarden** — отличный *личный* менеджер, но для команд неудобен из-за механики коллекций.
- **Teampass / SysPass / Passwordcockpit** — функциональны, но проигрывают в безопасности (нет E2E) и/или UX.
- **HashiCorp Vault** — мощный, но для другого сценария (инфраструктурные секреты). Нет браузерного расширения, сложное администрирование. Берите, если уже используете Vault в инфраструктуре.
- **KeePass-решения** — для команд требует коммерческого Pleasant Password Server.
- **gopass** — отличный CLI-менеджер для команд (git + GPG), но web-интерфейса нет — только браузерные расширения.
